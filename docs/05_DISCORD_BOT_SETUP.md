# 05. Discord Bot のセットアップ

## このセクションの目的

Discord Bot(ボット)を作成して、講座運営に使うDiscordサーバーに招待します。このBotは、Discordの「イベント」タブ(Scheduled Event)に講座開催予定を自動登録するために必要です。

「Bot」というと難しそうに聞こえますが、ここでやることは「Discord上で動くアシスタント役の専用アカウントを作って、自分のサーバーに住まわせる」というイメージです。プログラミングは必要ありません。画面の指示通りにクリックしていけば完了します。

## 所要時間

約15〜20分

## 前提

- Discordアカウントを持っていること
- 招待先のDiscordサーバーで「管理者」権限を持っていること(自分で作ったサーバーなら通常OK)
- ブラウザ(Chrome、Safariなど)が使えること

## 手順

### Step 1: Discord Developer Portal にアクセス

ブラウザで以下のURLを開きます。

```
https://discord.com/developers/applications
```

普段使っているDiscordアカウントでログインしてください(まだログインしていなければ求められます)。

「Applications(アプリケーション)」という一覧ページが表示されます。最初は空っぽです。

### Step 2: 新しいアプリケーションを作る

ページ右上にある青いボタン「**New Application**」をクリックします。

ポップアップが出てくるので:

1. 「**NAME**」欄にアプリの名前を入力します(例:`CourseBot`、`講座Bot` など、自分が分かれば何でもOK)
2. 利用規約のチェックボックスにチェックを入れます
3. 「**Create**」ボタンをクリック

これでアプリの管理画面が開きます。

### Step 3: Bot のトークンを取得する

左側のメニュー(サイドバー)から「**Bot**」をクリックします。

「Build-A-Bot」というセクションがあります。「**Reset Token**」というボタンをクリックしてください(初回の場合は「Copy」ボタンの場合もあります)。

確認ダイアログが出たら「**Yes, do it!**」をクリック。

長い文字列(Botのトークン)が表示されます。「**Copy**」ボタンでコピーしてください。

⚠️ **重要:このトークンは1回しか表示されません。**
ページを離れたり、再表示すると見えなくなります。今すぐ安全な場所(後述の `.env` ファイル)に貼ってください。万が一なくしたら、もう一度「Reset Token」してやり直しになります。

⚠️ **このトークンは絶対に他人に教えないでください。** Botを乗っ取られる原因になります。SNSやGitHubには絶対に貼らないこと。

### Step 4: Bot の権限設定(Intents)

同じ「Bot」ページを下にスクロールすると、「**Privileged Gateway Intents**」というセクションがあります。

以下の2つをONにしてください(クリックでトグル):

- **MESSAGE CONTENT INTENT**(メッセージ内容を読む権限)
- **SERVER MEMBERS INTENT**(サーバーメンバー情報を読む権限)

ページ下部の「**Save Changes**」ボタン(緑色)をクリックして保存します。

### Step 5: 招待用URLを生成する

左メニューから「**OAuth2**」をクリックします。

その中の「**URL Generator**」を開きます(クリックして展開)。

**Scopes(スコープ)** のセクションで、以下2つにチェック:

- `bot`
- `applications.commands`

すると下に「**Bot Permissions**」というセクションが追加で出てきます。以下にチェックを入れます:

- View Channels(チャンネル閲覧)
- Send Messages(メッセージ送信)
- Embed Links(リンクの埋め込み)
- Mention Everyone(@everyone でメンション)
- Create Events(イベント作成)
- Manage Events(イベント管理)

ページ下部に「**GENERATED URL**」という欄が出てきます。長いURL(`https://discord.com/api/oauth2/authorize?...` で始まる)が表示されているので、「**Copy**」ボタンでコピーします。

### Step 6: Bot をサーバーに招待する

コピーしたURLを、新しいブラウザタブで開きます。

「外部アプリケーションがDiscordアカウントへのアクセスを求めています」というページが出ます。

1. 「**サーバーに追加**」のプルダウンで、招待したいサーバーを選択
2. 「**はい**」をクリック
3. 権限の確認画面が出るので、内容を確認して「**認証**」をクリック
4. 「私はロボットではありません」のチェック(出た場合のみ)

⚠️ 招待には対象サーバーの **管理者権限** が必要です。権限がないと選択肢に出てきません。

### Step 7: `.env` ファイルにトークンを貼る

zoom-course-automation のフォルダにある `.env` ファイルを開きます(まだない場合は `.env.example` をコピーして作ります)。

```
cp .env.example .env
```

`.env` の中の以下の行を探して、Step 3でコピーしたトークンを貼ります:

```
DISCORD_BOT_TOKEN=ここにトークンを貼る
```

例:

```
DISCORD_BOT_TOKEN=MTIzNDU2Nzg5MDEyMzQ1Njc4OS5BYmNEZWY.GhIjKlMnOpQrStUvWxYz1234567890AbCdEfGh
```

ファイルを保存して閉じます。

## 動作確認

### こうなっていればOK

- Discord Developer Portal の Applications 一覧に、自分が作ったアプリ(例「CourseBot」)が表示されている
- 招待したDiscordサーバーの右側メンバー一覧に、Botの名前が表示されている(オフラインでも灰色でOK)
- `.env` ファイルの `DISCORD_BOT_TOKEN=` の右側に長い文字列が入っている

## トラブル時

### エラー: 「Missing Access」「Unknown Guild」などのエラーが後で出る

**原因**: BotをサーバーにまだInvite(招待)できていない、または権限が不足している。

**解決方法**: Step 5〜6をやり直して、必要な権限(特に「Create Events」「Manage Events」)が全部入っているか確認してください。一度招待し直すには、生成URLを再度開き直せばOK(同じBotが上書き招待されます)。

### エラー: トークンを保存し忘れた・どこかにいった

**原因**: トークンは1回しか表示されない仕様。

**解決方法**: もう一度 Bot ページの「**Reset Token**」をクリックして、新しいトークンを発行してください。古いトークンは無効になります(誰かに見られたかも、というときも Reset で安全になります)。

### Bot がサーバーに表示されない

**原因**: 招待URLを開いたときに、サーバー選択を間違えた。または、対象サーバーの管理権限がない。

**解決方法**: もう一度 Step 5 で生成したURLをブラウザで開き、正しいサーバーを選んで認証し直してください。

### MESSAGE CONTENT INTENT を ON にしようとしたら警告が出る

**原因**: Discordは大きなBot(100サーバー以上)では追加申請が必要というルールがあるため警告を出しますが、個人/小規模利用なら問題なくONにできます。

**解決方法**: 警告は無視してONにし、Save Changesをクリックして大丈夫です。
